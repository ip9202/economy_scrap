## 1. 프로젝트 개요

### 1.1 제품명
**RateStance** - 통화정책 뉴스 스탠스 분석 파이프라인

### 1.2 핵심 가치 제안
한국은행(BoK) 기준금리 결정과 금융 뉴스 감성(매파/비둘기파) 간의 관계를 자동으로 분석하는 데이터 파이프라인 및 대시보드.

### 1.3 해결하는 문제

| 문제 | 설명 |
|------|------|
| **정책 신호 포착 지연** | 기자·분석가가 수동으로 뉴스를 검토해야 해 시간이 많이 소요 |
| **히스토리 데이터 부재** | 과거 금리 결정 전후 여론 동향 파악 어려움 |
| **정량화 부재** | 뉴스 논조를 수치로 비교·추적하기 어려움 |

### 1.4 대상 사용자

| 사용자 | 주요 목적 |
|--------|-----------|
| **경제학 연구자** | 통화정책 기대와 미디어 보도 관계 연구 |
| **투자사 애널리스트** | 정책 방향 예측을 위한 감성 시그널 모니터링 |
| **중앙은행 연구원** | 시장의 정책 인식 분석 |
| **금융 미디어 편집자** | 보도 논조와 정책 결정 정합성 추적 |

## 2. 핵심 기능

### 2.1 이중 소스 뉴스 수집

**기간 분기 전략:**

| 기간 | 소스 | 방식 |
|------|------|------|
| 2025-08-01 이전 (과거) | **GDELT BigQuery** | BigQuery 공개 데이터셋 쿼리 |
| 2025-08-01 이후 (현재) | **Google News RSS** | RSS 피드 파싱 |

**수집 키워드 (기본값):**
- 한국어: "한국은행 기준금리", "통화정책", "금리"
- 영어: "Bank of Korea interest rate", "BOK rate", "Korea monetary policy"

**중복 제거 전략:**
1. `google_url` 정확 매칭
2. 정규화된 제목 + 발행일 조합

### 2.2 스탠스 스코어링

**알고리즘 (키워드 기반):**
```
stance_score = hawk_count - dove_count
```

| 구분 | 키워드 (기본값) |
|------|-----------------|
| **매파 (Hawkish)** | 인상, 긴축, 물가압력, 과열, 억제, 경계, 매파 |
| **비둘기파 (Dovish)** | 인하, 완화, 둔화, 부양, 하방, 지원, 비둘기 |

- `stance_score > 0`: 매파 (금리 인상 지지)
- `stance_score < 0`: 비둘기파 (금리 인하 지지)
- `stance_score == 0`: 중립

### 2.3 데이터 파이프라인

**6단계 순차 처리:**

```
[1] 뉴스 수집 (GDELT/RSS)
      ↓
[2] 스탠스 스코어링 (키워드 매칭)
      ↓
[3] 일별 집계 (날짜별 평균 스탠스)
      ↓
[4] 한국은행 기준금리 수집 (ECOS API)
      ↓
[5] 이벤트 감지 (금리 변동 분류: 인상/인하/동결)
      ↓
[6] 이벤트 스터디 (변동 전후 ±14일 윈도우 분석)
```

**출력 파일 (data/ 디렉토리):**
- `news_raw.csv` - 원본 뉴스
- `news_scored.csv` - 스탠스 스코어 포함
- `news_daily.csv` - 일별 집계
- `rate_series.csv` - 기준금리 시계열
- `events.csv` - 감지된 이벤트
- `event_study_table.csv` - 이벤트 스터디 결과

### 2.4 이벤트 스터디 분석

**방법론:**
- 각 금리 변동 이벤트를 기준으로 ±14일 윈도우 생성
- 윈도우 내 각 날짜의 뉴스 스탠스 데이터 매핑
- 이벤트 유형별(인상/인하/동결) 평균 스탠스 프로파일 생성

**연구 가설:**
- 사전 스탠스: 뉴스가 정책 결정을 선행하는가?
- 사후 반응: 시장이 인상/인하에 다르게 반응하는가?

### 2.5 대시보드 시각화

**메인 페이지 레이아웃:**

```
┌──────────────────────────────────────────────────────┐
│ 헤더: 스크래핑 버튼 + 데이터 새로고침 버튼             │
├──────────────────────────────────────────────────────┤
│ 통계 카드 4개: 총 기사 | 평균 스탠스 | 이벤트 | 최근 이벤트│
├──────────────────────────────────────────────────────┤
│ 날짜 범위 선택기 (1개월/3개월/6개월/1년/직접 지정)      │
├──────────────────────────────────────────────────────┤
│ 뉴스 스탠스 시계열 차트 (클릭 가능 - 기사 드릴다운)     │
├──────────────────────────────────────────────────────┤
│ 기준금리 시계열 차트                                   │
├──────────────────────────────────────────────────────┤
│ 이벤트 스터디 차트 (유형별 ±14일 프로파일)             │
└──────────────────────────────────────────────────────┘
```

**인터랙션:**
- 시계열 날짜 클릭 → 해당 날짜 ±3일 기사 목록 표시 (NewsDetailPanel)
- 이벤트 마커 클릭 → 이벤트 상세 정보 표시 (EventDetailPanel)
- 비동기 스크래핑 진행률 다이얼로그

## 3. API 스펙

### 기본 URL
- **Production:** `https://ip9202.site/api/` (포트 8001 → Nginx 프록시)
- **Development:** `http://localhost:8001/api/`

### 데이터 조회 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/data/news-daily` | 일별 뉴스 스탠스 집계 |
| GET | `/api/data/rate-series` | 기준금리 시계열 |
| GET | `/api/data/events` | 감지된 금리 변동 이벤트 |
| GET | `/api/data/event-study` | 이벤트 스터디 분석 결과 |
| GET | `/api/data/statistics` | 요약 통계 |
| GET | `/api/data/news-articles` | 개별 뉴스 기사 (페이지네이션) |
| GET | `/api/data/news-articles/by-date` | 특정 날짜 기사 조회 |

**공통 쿼리 파라미터:** `start_date`, `end_date` (YYYY-MM-DD)

### 데이터 갱신 엔드포인트 (비동기)

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/data/refresh` | 파이프라인 실행 (Job 생성) |
| GET | `/api/data/refresh/status/{job_id}` | Job 상태 조회 |
| GET | `/api/data/refresh/jobs` | 전체 Job 목록 |

**Job 상태 값:** `pending` → `running` → `completed` / `failed`

**진행률 스테이지:**
- 0-20%: 뉴스 수집
- 20-40%: 스탠스 스코어링
- 40-60%: 일별 집계
- 60-80%: 기준금리 수집
- 80-90%: 이벤트 감지
- 90-100%: 이벤트 스터디

## 9. 알려진 한계 및 향후 로드맵

### 현재 한계

| 한계 | 설명 | 완화 방법 |
|------|------|----------|
| **키워드 기반 감성만** | ML 모델 없음, 맥락 이해 불가 | 커스터마이즈 가능한 키워드 목록 |
| **단일 스레드 파이프라인** | 대용량 데이터 처리 느림 | 병렬 수집 단계 도입 가능 |
| **인메모리 Job 상태** | API 재시작 시 Job 상태 소멸 | DB 영속화 필요 |
| **인증 없음** | 모든 API 공개 | API 키 인증 추가 가능 |
| **한국어 뉴스 중심** | 국제 보도 미포함 | 영어 키워드 일부 포함됨 |

### 향후 기능 (로드맵)

1. **ML 기반 감성 분류** - BERT/GPT 기반 정확도 향상 (+15-20% 예상)
2. **데이터베이스 영속화** - PostgreSQL으로 확장성 확보
3. **실시간 스트리밍** - WebSocket 기반 업데이트 (폴링 대체)
4. **자동 스케줄링** - 일별 자동 파이프라인 실행
5. **다중 중앙은행 지원** - ECB, Fed, BoE 비교 분석
6. **Granger 인과성 검정** - 뉴스 선행성 통계 검증
